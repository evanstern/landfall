Debt, Tiers & the Empty Gate

How nobody starves, how the best affordable oracle wins — and why the gate contains no AI at all

The starvation problem

Picture a store that rain-checks you every time an item is sold out — and each refusal makes your voucher bigger. Refuse you often enough and eventually your voucher is so generous that even on a packed day, you get served. That’s suppression debt.

Why it’s needed: if the world stays hot for hours, the Gate keeps saying no, the oracle is never consulted, and your app flatlines at fallback quality forever. Debt guarantees an eventual yes:

debt 0 → budget 5.0 Predicted drift is 9 — refused. The class accrues one unit of debt.
debt 1 → budget 7.5 Budget 5 × (1 + 1 × 0.5) = 7.5. Still under 9 — refused again, debt grows.
debt 2 → budget 10.0 5 × (1 + 2 × 0.5) = 10 — drift 9 now fits. Admitted. A successful landing resets the debt to zero.
📈
The gate stays pure

Notice landfall doesn’t remember the debt — your app counts refusals and passes the number in, and the Verdict records it. Same inputs, same ruling, always. (This is the “fairness boost” you glimpsed in Module 1’s code.)

A ladder of oracles

Often you have several oracles for the same job: a brilliant slow cloud model, a decent quick local one. RouteTiered runs the gate against each and picks the best-quality tier whose answer would land in time:

🐋
Calm world → the big slow oracle

Plenty of budget headroom — take the smartest answer available.

🐇
Hot world → the small fast oracle

The deep answer would land stale; the quick one still lands true.

🦎
Nothing fits → the declared fallback

The reflex, template, or skip from Module 2 — always available, already declared.

route.go 

	chosen, ok := best, false
	for i, t := range tiers {
		v := Route(c, velocity, t.SecondsPerPoint, t.Debt)
		if !v.Allow {
			continue
		}
		if !ok || t.Quality > tiers[chosen].Quality {
			chosen, ok = i, true
		}
	}

Aha! Each tier keeps its own tab

Debt is per-tier: a quick local answer does not repay the debt owed to the suppressed cloud tier. In a persistently hot world, the cloud tier’s voucher keeps growing until a cloud call finally fits — so the system escapes not just “never consulted,” but “forever stuck on the cheap answer.”

The empty gate

Here’s the design decision that makes everything else trustworthy. The judge that governs the AI contains, deliberately, nothing:

🕰️

No clock

It never reads the time — every duration is handed in as a number.

🎲

No randomness

Same inputs, same verdict. Every time. Forever.

🚫

No model

No AI inside the gate — so the gate can’t hallucinate itself open, and no clever prompt injection can talk it open either.

🧾

No hidden state

Every ruling is reproducible from its own recorded fields — a property-based test re-derives each verdict from its own record to prove it.

🏛️
The gate that governs the model contains no model

This is the sentence to steal. When AI decides what to answer, something deterministic should decide whether to ask. Tell your AI tools: “keep the enforcement path pure — no model, no clock, no randomness in the code that says yes or no.”

Same reasoning behind one last refusal: budgets have no runtime tuning API. They are doctrine — constants a human reviews — because a system that quietly re-tunes its own limits is a model sneaking back into the enforcement path through the side door. The landing log (Module 3’s report card) tells humans when a constant is wrong.

Where this pattern travels

Nothing in the formula mentions AI. Any “fast world, slow expensive answer” problem fits:

📝

AI on live documents

Velocity = edits per second. Is the 20-second deep analysis worth it, or will the user type right through it?

📊

Trading

Velocity = market volatility; budget = how long an edge stays profitable. Identical formula.

🤖

Robotics

Heavy route planner vs. reactive controller — budget measured in how far the plan can diverge.

🧑‍💼

Human approvers

An approver is an oracle with hours of latency. Approval queues everywhere are full of stale asks nobody priced.

And that’s the whole machine: a pure gate that prices the bet, a lease that re-checks it at landing, an estimator that learns carefully, and debt that keeps anyone from starving — every decision leaving a receipt you can replay.
